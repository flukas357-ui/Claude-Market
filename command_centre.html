📖 How It Works

Two views — trade flow and full system map. Click any box for more detail.
⚡ How a trade is placed automatically
1
TradingView detects signal
BB Breakout v7.0 fires on daily candle close — sends BUY or SELL to webhook
instant
2
Webhook Server receives
claude-market.onrender.com/webhook — checks secret key, stores signal in memory
<1s
3
Claude AI validates
Scores signal 1–5 based on market conditions. Score below 3 = rejected automatically
1–2s
4
EA polls for signal
EA asks webhook every 10 seconds — receives symbol, action, score, signal type
max 10s
5
EA calculates trade parameters
Lot size = 2% risk · SL per asset (GOLD=100, BTC=200 pips) · TP = SL × 1.5 · Filling mode auto-detected
instant
6
Trade placed on MT5
OrderSend fired — ticket issued — SL and TP set automatically — email notification sent
instant
7
Trailing stop activates
L1: SL → entry (break-even) · L2: locks profit · L3: ATR trails behind price automatically
ongoing
Total time: signal → open trade
under 15 seconds
You do nothing — the system handles everything
🔑 API Keys — What Goes Where
You only manage 2 keys. Everything else is automatic.
ANTHROPIC_API_KEY
Created at: console.anthropic.com/settings/keys
→ Render Environment → GitHub Secrets
Starts with: sk-ant-api03-... · No spaces · Same key in both places
EMAIL_PASS (Gmail App Password)
Created at: myaccount.google.com → Security → App passwords
→ GitHub Secrets only
16 characters · Remove all spaces · NOT your normal Gmail password
No API key needed for:
TradingView — just webhook URL MT5 — just EA file Command Centre — reads webhook

🛡️ Trailing Stop Levels — Quick Reference
L1 — B/E Price up 50% of SL distance → SL moves to entry → zero loss guaranteed
L2 — LOCK Price up 100% → SL locks in 25% of expected profit
L3 — TRAIL Price up 150% → SL trails 1×ATR behind price continuously
